# REST API Catalogue (`/api/v1`) — Flutter‑ready

Cross‑cutting conventions (auth, envelope, pagination, errors, versioning) are defined in
[`../../architecture/08-api-architecture.md`](../../architecture/08-api-architecture.md). This
folder lists the **endpoints per module**. The machine‑readable **OpenAPI** spec is generated by
`dedoc/scramble` at build time (Phase 23) and exported as `openapi.json` for client generation.

- [`endpoints.md`](endpoints.md) — full endpoint table.

## Auth model (summary)
- `Authorization: Bearer <sanctum-token>` for authenticated calls.
- Public read endpoints (catalog browsing) work without a token; user‑contextual fields
  (`is_favourited`, `download_url`) are null/absent for guests.

## Build order
API routes are added **inside each feature phase** (so the mobile app can use them as soon as a
feature exists). Phase 23 only *hardens and documents* the surface — it does not invent new APIs.
